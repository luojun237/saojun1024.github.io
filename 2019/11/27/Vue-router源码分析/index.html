Vue-router源码分析

Vue-router 是 Vue 框架里一个重要的插件，基本上所有基于 Vue 开发的项目都离不开 Vue-router 的使用。使用 Vue-router 可以开发页面权限控制,菜单设计,缓存页面,路由守卫等功能。下面我将介绍 Vue-router 的核心源码。

Vue.use() 注册插件

在 Vue.js 中，插件定义比较广泛，例如 Vue 的原型方法、组件、mixin 混合、filters 管道都可以算的上插件, Vue 提供一个全局注册这些插件的API, 即 Vue.use()

Vue.use() 接收两个对象参数，第一个参数对象中必须要有名字为install方法，第二个参数为可选项，一般为一些额外配置, 上述例子在 obj 对象中提供了 install 方法，并且 Vue 规定了在 install 方法中我们可以获取到两个参数，一个即 Vue 的构造函数，一个为 Vue.use 传入的第二个参数。上述代码我们在install方法中拿到 Vue 对象之后在 Vue 的原型上绑定了一个$greeting() 方法,这样每个 Vue 页面实例都将可以使用这个方法。

所以要写一个开源的 Vue 插件最基本的方法是源码文件内导出一个含 install 方法的对象。

单页应用路由原理

在浏览器地址栏中输入地址 http://1.2.3.4:8080/ 这样的地址时，浏览器会根据这个地址去访问web服务器并返回相应内容。如果地址栏这个时候改变为 http://1.2.3.4:8080/a 时，浏览器认识到了地址发生改变，就会重新刷新页面获取新的内容,这个大家都知道。

但是地址由 http://1.2.3.4:8080/ 变为 http://1.2.3.4:8080/#a 时浏览器却不会重新刷新页面。我们把地址栏中带“#”后面部分称为 hash。如果只是 hash 改变了，浏览器不会刷新页面。我们可以使用原生的 JavaScript 获取到当前浏览器地址的 hash 部分:

我们还可以使用 JavaScript 监听到 hash 的改变,从而做一些事情

Vue-router 最基本的思路之一就是利用了 hashchange 事件。首先 Vue-router插件在use()时会全局注册一个组件RouterView,这个组件可以理解为一个“壳”,或者一个“舞台”,例如当我们输入地址 http://1.2.3.4:8080/#a 时, 这时 hash 名为 “#a”, Vue-router 会将 #a 对应的组件渲染到 RouterView 中来展示。当我们输入地址http://1.2.3.4:8080/#b 时这时触发 hashchange 事件,这个时候 RouterView 则渲染 #b 对应的组件。

实现 install 方法

在 Javascript 中一切皆对象,上述的构造函数 VueRouter 也不例外，所以我们可以给其加一个静态的 install 方法,有了install方法就可以称 VueRouter 是一个插件了。

当 Vue.use(VueRouter) 时会执行里面的 install 方法，首先会在 Vue 对象注册一个 mixin,这样 Vue 每个组件被加载时都会执行 beforeCreate,这里注意一定是组件被加载时，并不是install 方法执行时就会执行 beforeCreate, install 只会执行Vue.mixin()这个函数，而里面的 beforeCreate 只有组件加载时才会执行。

现在我们开始关注 beforeCreate 方法里面的逻辑，beforeCreate 第一次执行肯定是在根组件(main.js中的 new Vue)内执行的，因为根组件最先被加载.在根组件加载时我们需要在这个组件上声明一个属性_routerRoot用来存放当前实例信息，在接下来每个子组件加载时也声明一个属性_routerRoot去引用父一级的实例，这样最后每个组件内_routerRoot都共享的是一个对象，即根组件的 this ,至于为什么需要这样，我们后续再解释。
