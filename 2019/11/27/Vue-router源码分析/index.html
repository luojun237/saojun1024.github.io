Vue-router源码分析

Vue-router 是 Vue 框架里一个重要的插件，基本上所有基于 Vue 开发的项目都离不开 Vue-router的使用。使用 Vue-router 可以开发页面权限控制,菜单设计,缓存页面,路由守卫等功能。但也导致几乎前端难一点业务和Bug也容易出在这块。所以单单会用还是不行，下面我将介绍一下 Vue-router 的核心源码。

Vue.use() 注册插件

在Vue.js中，插件定义比较广泛，例如 Vue 的原型方法,组件,mixin混合,filters管道都可以算的上插件,Vue提供一个全局注册这些组件的API,即 Vue.use()

import Vue from 'vue'
const obj = {
  install:function(Vue,options){
    // 拿到了Vue对象和可选项目，就可以在这里面实现插件逻辑了
    Vue.prototype.$greeting= function(){
      console.log("hello")
    }
  }
}
const opt ={}
Vue.use(obj,opt)

Vue.use() 一般接收两个参数,第一个一般为一个对象，并且规定这个对象中必须有一个名字为install的方法。而 Vue 也约定了，只要有这个 install 的这个方法,则 install 会给你提供两个参数，一个即 Vue 对象，一个为在 Vue.use() 的第一个参数。接下来拿到这两个参数，就可以来做各种事情了。本质上，写 Vue 插件就是在写 install 方法里面的逻辑。
上述代码拿到Vue对象之后，我们在其原型上增加了一个方法$greeting,这样每个页面就都可以这样调用这个方法了：

this.$greeting()

所以要写一个开源的Vue插件最基本的方法是：
××.js源码文件

export default {
  install:function(Vue,options){
    // ...
  }
}

别人使用：

import Vue from 'vue'
import YourPlugins from '××.js'
Vue.use(YourPlugins)

单页应用路由原理

在浏览器地址栏中输入地址http://1.2.3.4:8080/这样的地址时，浏览器会去访问服务器的这个地址并根据服务器设置的返回类型而返回相应的内容。如果地址栏这个时候改变为http://1.2.3.4:8080/a时，浏览器认识到了地址发生改变，就会重新刷新页面获取新的内容。

但是地址由http://1.2.3.4:8080/变为http://1.2.3.4:8080/#a时浏览器却不会重新刷新页面。我们把地址栏中带“#”后面部分称为 hash。如果只是 hash 改变了，浏览器不会刷新页面。

我们可以使用原生的 JavaScript 获取到当前浏览器地址的 hash:

// 地址栏1 http://1.2.3.4:8080/#a
console.log(window.location.hash) // #a
// 地址栏2 http://localhost:8084/#quickStart/a
console.log(window.location.hash) // #quickStart/a
// 地址栏3 http://localhost:8084/#quickStart/#a
console.log(window.location.hash) // /#quickStart/#a

我们还可以使用 JavaScript 监听到 hash 的改变,从而做一些事情

window.addEventListener('hashchange',()=>{
  // 实现一些逻辑
})

Vue-router 最基本的思路之一就是利用了 hashchange 事件。首先 Vue-router 内置了一个组件RouterView,这个组件可以理解为一个“壳”,或者一个“舞台”,例如当我们输入地址http://1.2.3.4:8080/#a时,我们得知hash名为“a”, Vue-router 会将a对应的组件渲染到RouterView中来展示。当我们输入地址http://1.2.3.4:8080/#b时这时触发hashchange事件,这个时候 RouterView 则渲染 b 对应的组件。

<template>
  <div>
    <router-view/>
  </div>
</template>
