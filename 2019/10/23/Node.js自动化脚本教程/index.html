Node.js自动化脚本教程

真正的大牛和极客们，都不太乐于使用鼠标的操作。早些年写脚本多属于 Linux,PHP,Python 等工程师们的日常操作。随着 Node.js 的出现和发展, Node.js 具有了读写文件和操作底层系统的能力。前端工程师也可以写一些脚本来辅助日常工作。下面我会分享一些常见的 Node.js 脚本。

环境的搭建

前往 Node.js 官网下载并一路安装即可。再下载一个编辑器。这里推荐 vscode。当然你就是喜欢sublime,nodepad, 那我也没办法。。

在 windows 命令提示符下输入命令检验是否安装成功。

新建文件夹 node-script 并在cmd下进入该文件夹执行npm init -y

再该文件夹下新建一个文件index.js.开始我的第一个脚本

使用shelljs

shelljs基于 Node.js api封装了执行系统文件操作的命令,Github地址

  • 安装好shelljs

  • 第一个程序：

  • exec()执行命令

  • cat() 查看文件下的内容

  • mkdir() 创建文件夹

  • rm() 删除文件/文件夹

  • test(modify,path) 测试文件类型

修饰符 作用
-b true if path is a block device
-c true if path is a character device
-d true if path is a directory
-e true if path exists
-f true if path is a regular file
-L true if path is a symbolic link
-p true if path is a pipe (FIFO)
-S true if path is a socket

  • which() 检测是否可用某个命令

  • exit() 退出

  • cp() 复制

  • cd() 切换路径

  • ls() 列举目录下文件夹和文件

  • touch() 新建文件

  • sed() 文件内容查找与替换
