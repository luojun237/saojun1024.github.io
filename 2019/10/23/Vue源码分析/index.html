Vue.js源码分析。

做前端工作快三年了，目前已经能够熟练使用Vue进行常用的业务开发。如果还停留在只会使用Vue上那显然是不够的。下面我来简单地分析一下Vue原理，并写一个简单的MVVM框架。

有一个简单的html页面：

如果没有任何的框架,这就是一个原生的html的页面，大括号照常显示。现在我们要借助JavaScript能力来解析 name 和 age,对于这个需要解析的变量全部放到一个对象里进行管理，并且对象属性变化时，视图会自动进行刷新。下面我们就用对象(构造函数) Vue 来管理这些数据。定义一个基本的 Vue 对象，可以传入一个对象:

其中的 options.data 即为在 html 中需要使用和绑定的对象,当然可能有些属性，例如属性sex页面并没有使用到,这时我们需要进行特殊的处理，这个我们后面再讲。到这一步，其实我们就可以进行一个简单的编译，将双括号内容替换成age,将input的value值设置成v-model绑定的值。我们可以采用纯粹的DOM操作方式，但是如果有值更新了，我们又要操作一遍DOM,这样就会有性能损耗。为了优化，我们可以创立文档碎片的方式来进行。先将vue实例根节点的el转换成文档碎片fragment进行解析编译操作，解析完成，再将fragment添加回原来的真实dom节点中。

可以理解为fragment是一个小纸条，可以将dom树中的子元素加入进来，fragment存在于内存之中，并不会引起重绘等问题。在需要时再将这个片段塞回去文档之中，即由原来的频繁操作改为一次性插入。
